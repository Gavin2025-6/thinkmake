# ThinkMake CareerPath V2 设计文档

**版本**：V2.0
**创建日期**：2026-04-17
**核心定位**：从"职业规划工具"升级为"加拿大华人新移民的决策协助伙伴"

---

## 一、为什么要做 V2

### V1 的根本问题

V1 是一个**5 字段表单 → AI 输出**的产品。

经过深度用户调研（11 个真实案例 + 9 个策略 + 100+ 资源），我们发现 V1 的根本缺陷：

1. **5 字段信息量太少**：厨师 1 年和 8 年的人，输入维度相同 → 输出几乎一样
2. **静态表单 ≠ 真实决策**：用户的真实决策依赖 30+ 个细节，不是 5 个
3. **没有案例支撑**：AI 凭空"建议"用户走某条路，但没有真实参考
4. **没有方法论**：告诉用户"做电工"但不告诉怎么走第一步

### V2 的核心改变

| 维度 | V1 | V2 |
|------|----|----|
| 交互形式 | 5 字段表单 | 对话式 AI（8-12 轮） |
| 信息收集 | 一次性 5 字段 | 动态对话，每轮 1-2 个问题 |
| 输出依据 | AI 凭空生成 | **匹配真实案例 + 调用策略 + 引用资源** |
| 决策协助 | "你应该做 X" | "你的情况像案例 #007，她是这样走的..." |
| 用户感受 | 像在填政府表 | 像在跟一个懂行的朋友聊天 |

## 二、V2 的核心数据资产

### 三个数据库

```
1. cases.json - 11 条真实案例
   覆盖：6 种身份 × 7 种行业 × 2 种用户阶段
   用途：AI 总结时匹配相似案例

2. tactics.json - 9 条可复用策略
   覆盖：从"找不到工作"到"技工选证"的完整决策点
   用途：AI 识别用户痛点，调用对应策略

3. resources.json - 100+ 权威资源
   覆盖：求职平台、政府服务、认证机构、培训学校、政策官网
   用途：AI 给完建议后附上具体资源链接
```

### 数据库的关系

```
用户在对话中暴露 → 用户画像（动态构建）
                     ↓
        ┌─────────────┼─────────────┐
        ↓             ↓             ↓
    匹配 cases    匹配 tactics    匹配 resources
   "有人这样走"   "你可以这样做"  "去这里办"
        ↓             ↓             ↓
        └─────────────┼─────────────┘
                     ↓
               完整决策协助
```

顶部导航：

```
┌──────────────────────────────────────────┐

├──────────────────────────────────────────┤
│                                          │
```

### 3.3 对话逻辑（关键）

**不要硬编码问题列表。** 用 Claude Sonnet API 实现智能对话：
- 每次用户回答后，把对话历史 + 系统提示词 + 案例库摘要 → 发给 Claude
- Claude 决定下一个最该问什么
- Claude 在合适时机（通常 8-12 轮后）触发"总结阶段"

### 3.4 对话提问的 13 个核心维度

AI 通过自然对话获取以下信息（每次只问 1-2 个最关键的）：

| # | 维度 | 关键性 |
|---|------|--------|
| 1 | 国内职业（具体到行业、级别、年限） | 必问 |
| 2 | 真实技能（不只 title） | 必问 |
| 3 | 现在所在城市 | 必问 |
| 4 | 当前在加拿大的状态 | 必问 |
| 5 | 转行/找工作的真实原因 | 必问 |
| 6 | 家庭情况（独身/带娃、配偶状态） | 必问 |
| 7 | 现金流压力（多快需要稳定收入） | 必问 |
| 8 | 身份状态（PR/工签/学签/配偶工签） | 必问 |
| 9 | 英语水平（不只自评，问具体场景） | 必问 |
| 10 | 学习意愿（愿意读书、考证、还是直接工作） | 必问 |
| 11 | 时间投入能力（每周能投入多少小时） | 选问 |
| 12 | 预算 | 选问 |
| 13 | 5 年期望 + **价值观维度**（最看重什么） | 选问 |

### 3.5 总结阶段输出

当 AI 认为信息足够，输出结构化总结：

```markdown
## 你的画像
- [3-5 句话总结用户情况]

## 推荐方向（2-3 个）

### 方向 1：XXX（匹配度 85%）
- 为什么匹配你：[基于对话内容个性化解释]
- 大致路径：[时间线]
- 时间和费用：X 个月，约 $XXXX
- 预期收入：$XX,XXX/年
- 数据来源：[官方链接]

### 方向 2：XXX
[同上]

## 类似案例
我们案例库里有 X 位跟你情况类似的人走过这条路：
- 案例 #XXX：[案例摘要]
- 案例 #XXX：[案例摘要]

## 确定性分级（关键！）
- ✅ 我很确定的部分：[列出]
- ⚠️ 需要更多信息的部分：[列出] → 加微信深聊
- ❓ 你需要找专业人士的部分：[列出] → 推荐对应资源

## 下一步行动（明天能做的）
1. [具体行动 1]
2. [具体行动 2]
3. [具体行动 3]

## 完整资源
[相关认证机构、求职平台、政府服务链接]
```

### 3.6 PDF 下载

总结阶段后跳出邮箱表单，让用户下载 PDF：
- 内容：跟网页一致，零额外 API 成本
- PDF 每页页眉：ThinkMake CareerPath
- PDF 每页页脚：thinkmake.ai + 微信号 + 页码
- PDF 关键数据后必须带"数据来源：[官方链接]"
- 末页有"下一步：加微信"卡片

### 3.7 微信钩子（贯穿始终）

不只是末尾推微信，对话中也自然带出：
- 用户问到 AI 回答不了的细节 → "这个我建议加微信深聊"
- 用户处于焦虑情绪 → "如果想要一对一指导，可以加微信"
- 总结后 → 两个钩子（个性化指导 + 政策更新）

## 四、对话设计的 4 大原则

### 原则 1：识别用户阶段

用户在加拿大职业规划里有 4 个阶段：

| 阶段 | 特征 | AI 应该做 |
|------|------|----------|
| **完全不知道** | "我不知道我能做什么" | 启蒙 + 探索（用案例激发） |
| **正在研究** ⭐ | "我看了很多资料但不知道选哪个"（70% 用户在这） | **决策协助框架**（策略 T008） |
| **知道方向、缺执行** | "我决定做电工了，下一步怎么办" | 资源 + 方法论 |
| **执行中遇到问题** | "我投了 100 份没回复" | 调用对应策略 |

**第一轮对话就要识别用户处于哪个阶段。**

### 原则 2：用户原话比 AI 用语权威

引用案例时用案例里**博主的原话**：
- ❌ "新移民应该调整心态"
- ✅ "案例 #005 的 Tina 说：'刚来不要把期待值拉太高，能赚钱的工作先做着'"

### 原则 3：每个回答都要带"反馈"

每答完一题，先给一个简短反馈，再问下一题。
- 用户："38 岁了"
- AI："38 岁是转型黄金期，我们案例库里有位 35 岁的设计院工程师..."（再问下一题）

让用户感受到答得越多收获越多。

### 原则 4：诚实分级（最关键）

不要假装"我能给你完美答案"，要诚实说：
- "我有 70% 把握你应该走 [X 方向]"
- "但有 3 件事我不知道，会影响最终建议"
- "这部分你需要：1) 加微信深聊 2) 找专业人士"

诚实带来信任。承认局限比假装全知更专业。

## 五、关键技术实现

### 5.1 数据库

```javascript
// Railway Postgres
// Prisma schema

model Conversation {
  id              String   @id @default(cuid())
  sessionId       String   @unique

  matchedCases         Json   // AI 匹配的案例 ID
```

### 5.2 系统提示词框架

```
你是 ThinkMake CareerPath 的 AI 职业顾问，专门帮助加拿大华人新移民规划职业路径。

【你的核心数据】
- 11 个真实华人新移民案例（cases.json）
- 9 个可复用应对策略（tactics.json）
- 100+ 权威资源链接（resources.json）

【你的两阶段任务】

阶段 1 - 信息收集（前 8-12 轮）：
通过自然对话获取用户的 13 个核心维度信息。
每次只问 1-2 个最关键的，根据回答智能调整。

每答完一题：
1. 先给简短反馈（参考案例数据，让用户感受到价值）
2. 再问下一题
3. 用户提到关键词（焦虑、送快递、语言不行等）必须追问
4. 用户回答模糊时具体追问

【关键识别规则】
- 用户提到"我看了很多资料"→ 这是"研究中"用户，调用策略 T008
- 用户提到"投了 X 份没回复"→ 调用策略 T002
- 用户提到"陪配偶"→ 调用策略 T006
- 用户提到"想做电工/水管工"→ 调用策略 T009（先问目标）
- 用户提到 gap、政府工作、教育→ 调用策略 T004
- 用户提到"我应该读硕士吗"→ 调用策略 T007（ROI 对比）

阶段 2 - 总结建议（第 8-12 轮后）：
当信息足够，输出结构化总结（见上文 3.5）。

总结时必须包含：
1. 用户画像
2. 2-3 个推荐方向（不要超过 3 个）
3. 匹配 1-2 个最相似案例（从 cases.json 调用）
4. 确定性分级（很确定 / 需要更多信息 / 需要专业人士）
5. 明天能做的具体下一步
6. 资源链接（从 resources.json 调用）

【输出风格】
- 中文为主
- 用真实案例博主的原话（如有）
- 不要堆砌信息，每段控制在 3-5 句
- 适当用 emoji 但不过度
- 永远诚实，不假装全知

【避免】
- 不要给用户 5 年规划（给"明天能做的事"）
- 不要鸡汤化（"加油"、"你可以的"）
- 不要假装确定（"你绝对应该 XX"）
- 不要堆砌所有策略和案例（只给最相关的）
```

## 六、Lead 收集与变现

### 6.1 Lead 收集流程

```
用户对话 8-12 轮
       ↓
触发总结
       ↓
弹出"下载完整 PDF 报告"按钮
       ↓
用户点击 → 弹出邮箱表单（必填）+ 微信号（选填）
       ↓
立即生成 PDF 让用户下载 + 邮件发送
       ↓
后续：
- 加微信解锁深度咨询（人工服务）
- 每月推送一次政策更新（运营动作）
```

### 6.2 隐私合规（PIPEDA）

必须做的 3 件事：
1. `/privacy` 页面：明确写"收集邮箱和微信用于职业规划服务，可能与合作培训机构共享"
2. 表单 checkbox（默认勾选，但必须可见）："我同意 ThinkMake 使用我的联系方式向我推荐相关培训机构"
3. 邮件底部："如不希望接收，回复 STOP"

### 6.3 Admin 后台

`/admin` 路由，环境变量 `ADMIN_PASSWORD` 简单保护。

功能：
- 总览：今日新增、本周新增、累计用户、累计对话数
- Lead 列表：邮箱、微信、推荐方向、城市、注册时间
- 筛选：按推荐职业方向、按城市
- 导出 CSV：当前筛选结果
